# CUA Modal Watcher

This folder contains a **Computer-Using Agent (CUA)** that:
1. Watches for Excel VBE runtime error modals and auto-dismisses them
2. **Reads the yellow and blue Codex token counters on every poll** so the session
   state file always has a fresh token budget snapshot for Codex to reference

## Token Color Conventions

The Codex panel in VS Code shows two token counters:

| Color | Meaning | Action |
|---|---|---|
| 🟡 **Yellow** | Context tokens **remaining** — warning indicator, getting close to the limit | If low, Codex should summarize/compress context before continuing |
| 🔵 **Blue** | Tokens **used** so far in this session | Informational — tells you how deep into the session you are |

The watcher captures both on **every** 2-second poll and writes them to
`xlam_session_status.txt` under a `token_snapshot` key, even when no modal
is present. On a modal dismissal event, they are also written to
`build_xlam_run.log` and `cua_last_event.txt`.

## Files

| File | Purpose |
|---|---|
| `cua_modal_watcher.py` | Main loop — screenshot → vision (modal + tokens) → click + status update |
| `cua_codex_bridge.py` | Reads session state files and formats a ready-to-paste Codex prompt |
| `codex_prompt_draft.txt` | Auto-generated by bridge — paste into Codex after a dismissal event |

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│  cua_modal_watcher.py  (runs in background)                     │
│                                                                 │
│  Every 2s:                                                      │
│    1. Screenshot full screen                                    │
│    2. Ask GPT-4o (single call):                                 │
│       a. Is there an Excel VBE error modal?                     │
│       b. What are the yellow (remaining) token count?           │
│       c. What is the blue (used) token count?                   │
│                                                                 │
│    IF modal detected:                                           │
│       3. Locate 'End' button via vision coords                  │
│       4. pyautogui.click(x, y)  ← modal dismissed              │
│       5. Write xlam_session_status.txt  (full event + tokens)   │
│       6. Append to build_xlam_run.log                           │
│       7. Write cua_last_event.txt  ← Codex bridge reads this    │
│                                                                 │
│    IF no modal (normal poll):                                   │
│       3. Update token_snapshot in xlam_session_status.txt only  │
│          (yellow + blue always stay current)                    │
└─────────────────────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────────┐
│  cua_codex_bridge.py  (run manually after a dismissal event)    │
│                                                                 │
│    Reads:  xlam_session_status.txt + cua_last_event.txt         │
│    Writes: codex_prompt_draft.txt                               │
│    Output: Formatted prompt (incl. token budget) → paste Codex  │
└─────────────────────────────────────────────────────────────────┘
```

## xlam_session_status.txt Schema

### On a clean poll (no modal)
```json
{
  "token_snapshot": {
    "timestamp": "2026-04-01T13:00:00",
    "yellow_remaining": "14,203",
    "blue_used": "83,451"
  }
}
```

### On a modal dismissal event
```json
{
  "timestamp": "2026-04-01T13:05:12",
  "event": "vba_modal_dismissed",
  "error": "Run-time error '91': Object variable or With block variable not set",
  "button_clicked": "End",
  "status": "excel_unblocked",
  "tokens": {
    "yellow_remaining": "14,203",
    "blue_used": "83,451"
  },
  "token_snapshot": {
    "timestamp": "2026-04-01T13:05:12",
    "yellow_remaining": "14,203",
    "blue_used": "83,451"
  }
}
```

## Setup

```bash
# Install dependencies
pip install openai pillow pyautogui pygetwindow

# Set your OpenAI API key
$env:OPENAI_API_KEY = "sk-..."

# Start the watcher (keep this running in a background terminal)
python tools/cua/cua_modal_watcher.py

# After an Excel modal is auto-dismissed, run the bridge and paste into Codex
python tools/cua/cua_codex_bridge.py
```

## Why CUA Instead of UIAutomation / AHK

| Approach | Problem | CUA Fix |
|---|---|---|
| AutoHotkey | Misses async VBA runtime modals | Vision loop catches any modal regardless of timing |
| PowerShell UIAutomation | COM/accessibility tree is **frozen** during modal-loop state | CUA bypasses the API — works at pixel + input level |
| **CUA (this tool)** | ✅ None — operates below the process API layer | GPT-4o reads the screen; pyautogui clicks the button; tokens captured every poll |
