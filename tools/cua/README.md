# CUA Modal Watcher

This folder contains a **Computer-Using Agent (CUA)** that watches for Excel VBE runtime error modals and automatically dismisses them so Codex can continue the invSys XLAM build uninterrupted.

## Files

| File | Purpose |
|---|---|
| `cua_modal_watcher.py` | Main loop — screenshots + vision detection + button click + status update |
| `cua_codex_bridge.py` | Reads session state files and formats a ready-to-paste Codex prompt |
| `codex_prompt_draft.txt` | Auto-generated by bridge — paste this into Codex after a dismissal event |

## How It Works

```
┌─────────────────────────────────────────────────────────────┐
│  cua_modal_watcher.py  (runs in background)                 │
│                                                             │
│  Every 2s:                                                  │
│    1. Screenshot full screen                                │
│    2. Ask GPT-4o: "Is there an Excel error modal?"          │
│    3. If yes → locate 'End' button via vision               │
│    4. pyautogui.click(x, y)  ← modal dismissed              │
│    5. Write xlam_session_status.txt  ← Codex reads this     │
│    6. Append to build_xlam_run.log                          │
│    7. Write cua_last_event.txt                              │
└─────────────────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│  cua_codex_bridge.py  (run manually after event)            │
│                                                             │
│    Reads:  xlam_session_status.txt + cua_last_event.txt     │
│    Writes: codex_prompt_draft.txt                           │
│    Output: Formatted prompt ready to paste into Codex chat  │
└─────────────────────────────────────────────────────────────┘
```

## Setup

```bash
# Install dependencies
pip install openai pillow pyautogui pygetwindow

# Set your OpenAI API key
$env:OPENAI_API_KEY = "sk-..."

# Start the watcher (keep this running in a background terminal)
python tools/cua/cua_modal_watcher.py

# After an Excel modal is auto-dismissed, run the bridge
python tools/cua/cua_codex_bridge.py
# Then paste tools/cua/codex_prompt_draft.txt into Codex
```

## Why CUA Instead of UIAutomation / AHK

| Approach | Problem | CUA Fix |
|---|---|---|
| AutoHotkey | Misses async VBA runtime modals | Vision loop catches any modal regardless of timing |
| PowerShell UIAutomation | COM/accessibility tree is **frozen** during modal-loop state | CUA bypasses the API — works at pixel + input level |
| **CUA (this tool)** | ✅ None — operates below the process API layer | GPT-4o reads the screen; pyautogui clicks the button |

## Integration with xlam_session_status.txt

The watcher writes a JSON object to `xlam_session_status.txt` on every dismissal:

```json
{
  "timestamp": "2026-04-01T11:00:00",
  "event": "vba_modal_dismissed",
  "error": "Run-time error '91': Object variable or With block variable not set",
  "button_clicked": "End",
  "status": "excel_unblocked"
}
```

Codex can poll this file to know when Excel is clear and it's safe to resume.
