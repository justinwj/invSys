# CUA Modal Watcher

This folder contains a **Computer-Using Agent (CUA)** that:
1. Watches for Excel VBE runtime error modals and auto-dismisses them
2. **Reads the VBE code editor highlight colors before dismissing** so Codex
   knows exactly which line caused the fault
3. Logs everything to `xlam_session_status.txt` and `build_xlam_run.log`

## VBE Highlight Color Conventions

When a VBA error occurs, the Visual Basic Editor highlights code lines with
two distinct colors inside the code pane:

| Color | VBE Meaning | What CUA captures |
|---|---|---|
| 🟡 **Yellow** | The line where **execution stopped** — the actual fault line | Exact code text + line number |
| 🔵 **Blue** | **Breakpoint** lines — lines the developer marked for step-debugging | List of all blue-highlighted code lines |

The yellow line is the most important: it tells Codex **exactly what code
statement failed**, without having to guess from the error message alone.

## Files

| File | Purpose |
|---|---|
| `cua_modal_watcher.py` | Main loop — screenshot → vision (modal + yellow line + blue lines) → click + status update |
| `cua_codex_bridge.py` | Reads session state files and formats a ready-to-paste Codex prompt |
| `codex_prompt_draft.txt` | Auto-generated by bridge — paste into Codex after a dismissal event |

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│  cua_modal_watcher.py  (runs in background)                     │
│                                                                 │
│  Every 2s:                                                      │
│    1. Screenshot full screen                                    │
│    2. Ask GPT-4o (single call):                                 │
│       a. Is there an Excel VBE error modal dialog?              │
│       b. What line is YELLOW in the VBE code pane?              │
│          (= the fault line where execution stopped)             │
│       c. What lines are BLUE in the VBE code pane?              │
│          (= developer breakpoints)                              │
│                                                                 │
│    IF modal detected:                                           │
│       3. Locate 'End' button via vision coords                  │
│       4. pyautogui.click(x, y)  ← modal dismissed              │
│       5. Write xlam_session_status.txt  (error + highlights)    │
│       6. Append to build_xlam_run.log                           │
│       7. Write cua_last_event.txt  ← Codex bridge reads this    │
└─────────────────────────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────────┐
│  cua_codex_bridge.py  (run manually after a dismissal event)    │
│                                                                 │
│    Reads:  xlam_session_status.txt + cua_last_event.txt         │
│    Writes: codex_prompt_draft.txt                               │
│    Output: Formatted prompt (error + fault line) → paste Codex  │
└─────────────────────────────────────────────────────────────────┘
```

## xlam_session_status.txt Schema (on dismissal)

```json
{
  "timestamp": "2026-04-01T13:05:12",
  "event": "vba_modal_dismissed",
  "error": "Run-time error '91': Object variable or With block variable not set",
  "button_clicked": "End",
  "status": "excel_unblocked",
  "vbe_highlights": {
    "yellow_fault_line":     "Set ws = ThisWorkbook.Sheets(sheetName)",
    "yellow_fault_line_num": "42",
    "blue_breakpoint_lines": [
      "If Not ws Is Nothing Then",
      "ws.Range(\"A1\").Value = itemCode"
    ]
  }
}
```

## Setup

```bash
pip install openai pillow pyautogui pygetwindow
$env:OPENAI_API_KEY = "sk-..."

# Keep running in a background terminal while you work
python tools/cua/cua_modal_watcher.py

# After any auto-dismissed modal, generate the Codex prompt
python tools/cua/cua_codex_bridge.py
# Then paste tools/cua/codex_prompt_draft.txt into Codex
```

## Why CUA Instead of UIAutomation / AHK

| Approach | Problem | CUA Fix |
|---|---|---|
| AutoHotkey | Can dismiss the modal but can't read the yellow/blue code highlights | Vision reads the VBE editor pane directly |
| PowerShell UIAutomation | COM tree frozen during modal-loop; can't inspect VBE editor state | CUA bypasses APIs entirely — pixel level |
| **CUA (this tool)** | ✅ None | GPT-4o reads modal text + yellow fault line + blue breakpoints in one shot |
