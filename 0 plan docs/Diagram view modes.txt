# üúÅ **1. System-Level Views** (5 views)

These explain *the whole system* at high altitude.

1. **Ecosystem Map**

   * Systems, subsystems, external dependencies.

2. **Service Boundary / Module Boundary View**

   * Major modules/components + directed dependencies.

3. **Deployment & Hosting View**

   * Where things run. Machines, containers, Excel, clients.

4. **Integration Interfaces View**

   * APIs, data imports/exports, scheduled tasks.

5. **Runtime Behavior Overview**

   * High-level diagram of how the system behaves at runtime (e.g., request paths or event flows).

---

# üúÇ **2. Architectural Views** (8 views)

These explain *the architecture of the codebase* itself.

6. **Layered Architecture View**

   * UI ‚Üí Logic ‚Üí Data ‚Üí Integration.

7. **Module Internal Structure View**

   * What's inside a module (submodules, classes, forms).

8. **Component Communication View**

   * Major flows between high-level components.

9. **Data Architecture View**

   * All major datasets, tables, files, logs, bindings.

10. **Error & Logging Architecture View**

* How errors propagate and where logs are stored.

11. **Security / Permissions View**

* Roles, access flows, protected areas.

12. **Configuration View**

* App settings, environment variables, workbook/names-defined config.

13. **State-Transition View**

* How the system changes state during key operations.

# üúÇ **6. Code-Level / Developer-Facing Views** (10 views)

32. **Call Graph View (filtered)**

* Per module only, never full system.

33. **Event Graph**

* Events ‚Üí handlers.

34. **Module Dependency View**

* Submodules / functions calling each other.

35. **Error Propagation View**

* Try/catch boundaries, error routes.

36. **Function Responsibility Map**

* What each function *does* or *touches*.

37. **Hotspot View**

* What code changes most frequently.

38. **Anchor Classes / Core Routines View**

* Central routines and who they depend on.

39. **Threading / Asynchrony View** (if relevant)

* For VBA/Excel this is usually trivial or unused.

40. **Control Flow Graph (CFG)**

* High granularity: branches, loops, exits.

41. **Data Structure View**

* Types + fields + relationships.
